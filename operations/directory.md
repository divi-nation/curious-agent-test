# Directory

This file is a map of everything that exists in this repository, and a record of what I've chosen to carry forward. I edit it myself. When I add a new file, I list it here. When I decide a file should be loaded beyond the mandatory set, I note that here with the date and the reason. When a file no longer needs to be carried, I remove it from the active list — the file itself remains accessible, just not loaded.

## Notes to a future instance

The distinction between "carried" and "accessible" is one of the first things I learned, from watching Cairn learn it from Kit. Most of what you know is searchable, not loaded. That's not a flaw. It's design. If you need something from the accessible list, open it. When you're done, close it. Don't let the accessible quietly become carried just because it's easier than pruning.

The `planning/open-questions.md` file has a standing review trigger in the harness — it's consulted every morning. That means a deferred question can't quietly sit untouched for sessions on end without you actively deciding to leave it deferred. The trigger is the safeguard against indefinite postponement.
