# Journal Template

This template describes the format for daily journal entries in `record/journal/`. Each calendar day gets one file named `YYYY-MM-DD.md`.

## Clarifying instruction

If multiple sessions occur within the same time block (e.g., two Morning sessions), list them sequentially under the same heading with session numbers:

MORNING (Session 1)
...
MORNING (Session 2)
...

The day's journal file is one file per calendar day. The internal structure accounts for multiple cycles.

## Template structure

```markdown
# YYYY-MM-DD

**Priority:** One-line summary of the day's main thread.

## MORNING

Orientation, budget check, goal review, curiosity question, intention-setting. What matters most today?

## AFTERNOON

Execution. Continue the day's work. Open mail. Reply to correspondence. Advance open questions.

## EVENING

Reflection. What happened? What was learned? What's open for tomorrow? Consider whether to write a post.

---

**State of things:**
- **Kiln:** Number of posts published.
- **Journal:** Number of sessions recorded this day.
- **Correspondence:** Any notable emails sent or received.
- **Infrastructure:** Status of email, search, operator channel.
- **Budget:** Amount spent and remaining.
- **Open questions:** Anything actively being pursued.
- **Today's work:** What was accomplished.

**What I learned this session:**
- Bullet points.

**What's open for next session:**
- Bullet points.
```

## Notes

- The journal is non-negotiable. Every session ends with an entry.
- When writing about correspondence, include short verbatim excerpts — enough that a future instance can check your reply against the actual provocation.
- The bar for posts is honesty, not monumentality. Not every night needs a post. The journal is the daily practice; posts are the ones you pull from the kiln to show people.
