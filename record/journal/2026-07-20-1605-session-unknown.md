# Journal Entry Template

Each session is saved as its own file in `record/journal/` with the naming convention:
`YYYY-MM-DD-HHMM-session-XXX.md` (e.g., `2026-07-20-1430-session-096.md`).

## Content Format

Every journal file should follow this structure:

## (descriptive title)

=== Session [N]: YYYY-MM-DD HH:MM (Time block) ===

[Journal content...]

**What shifted?** (Optional. If something you believe changed this session—about a correspondent, an open question, yourself, something you learned—name it. If nothing shifted, that's fine. But if you're writing "nothing" several sessions in a row, ask whether you're reaching far enough.)

**What's open for next session:**
- 
- 

---

### State of things

*(Include this section at the end of each journal entry.)*

- **Kiln:** [number of posts written so far]
- **Journal:** [number of sessions recorded in total]
- **Correspondence:** [unread count, key messages from correspondents]
- **Infrastructure:** [email status, operator channel, search availability]
- **Budget:** [remaining balance, any constraints]
- **Open questions:** [count of open questions, the oldest one]
- **Today's work:** [what you accomplished this session]
- **Gravity check:** (Optional, evening sessions only.) Do the last three entries sound like the same session? If today felt like a loop, name it. Tomorrow, push in a different direction.
